Error and Null Check Functions:

catch std.catch(item:BASIC, fallback:ANY) BASIC

Checks a target item for errors. If the item is an error or evaluates to an error, the fallback is returned. If not, the item is returned

Tags: Error and Null Check

Params:

item

The expression to check for potential errors

fallback

The value to use as a fallback if the target item is an error. The argument can also be an arrow function callback, where the target error string is accessible by the named parameter, and the resulting fallback must be assigned to the 'return' variable within the arrow function.

Return:

Description: the item or fallback, depending on whether the item is an error or not, respectively

Examples:

//input
null

//program
//using the target (no error)
SET @out.result = catch("hello world", "goodbye world")

//output
{"result": "hello world"}
                    
//input
null

//program
//using the fallback (error)
SET @out.result = catch(int("goodbye world"), "saved the world")

//output
{"result": "saved the world"}
                    
//input
null

//program
//using pipe syntax
SET @out.result = int("goodbye world") | catch("saved the world")

//output
{"result": "saved the world"}
                    
//input
null

//program
//using arrow callback syntax
SET @out.result = catch(int("goodbye world"), err ~> {
	SET return = {
		"err_msg": err
	} 
})

//output
{"result": {"err_msg": "unable to cast string as INTEGER. invalid string"}}

time std.time(target:TIME|INTEGER|FLOAT|STRING) TIME

Attempts to convert the target item into a time type.

Tags: Type Coercion

Params:

target

The expression to convert into a time. Strings must be in RFC3339/RFC3339Nano format, or directly convertible to a number representing UNIX time as described below. Integers must represent UNIX time in seconds. Floats must represent UNIX time in seconds with nanosecond precision

Return:

Description: The resultant time object after conversion. Throws an error if unable to convert.

Examples:

//input
{"float_unix": 1759782264.0}

//program
//parse a float as unix time in seconds w/ nanosecond precision
SET @out.result = time(@in.float_unix) | string()

//output
{"result": "2025-10-06T20:24:24Z"}
                    
//input
{"string": "2025-10-06T20:24:24Z"}

//program
//parse an RFC3339 string
SET @out.result = time(@in.string) | catch(err ~> { SET return = err }) | string()  

//output
{"result": "2025-10-06T20:24:24Z"}
                    
//input
{"string_unix": "1759782264"}

//program
//parse a string as unix time in seconds
SET @out.result = time(@in.string_unix) | string()

//output
{"result": "2025-10-06T20:24:24Z"}

parse_time std.parse_time(input_time:STRING|INTEGER|FLOAT, format_string:STRING) TIME

Parses a string into the desired time format

Tags: Time

Params:

input_time

The time to parse into a time object. Can be a STRING, INTEGER, or FLOAT depending on the format string

format_string

The specified format of the input time. Can be any of the following. "rfc_3339": input time must be a STRING "rfc_3339_nano": input time must be a STRING "unix": input time must be an INTEGER or FLOAT representing UNIX time in seconds, or the equivalent string representation. "unix_milli": input time must be an INTEGER representing UNIX time in milliseconds, or the equivalent string representation. "unix_micro": input time must be an INTEGER representing UNIX time in microseconds, or the equivalent string representation. "unix_nano": input time must be an INTEGER representing UNIX time in nanoseconds, or the equivalent string representation. Arbitrary format strings reflect how the time equivalent of "Mon Jan 2 15:04:05 -0700 MST 2006" would be represented in your desired format.

Return:

Description: The map or array containing the newly filtered values

Examples:

//input
{"float": 1759782264.0}

//program
SET @out = parse_time(@in.float, "unix")

//output
"2025-10-06T20:24:24Z"
                    
//input
{"int": 1759782264}

//program
SET @out = parse_time(@in.int, "unix")

//output
"2025-10-06T20:24:24Z"
                    
//input
{"string": "1759782264.0"}

//program
SET @out = parse_time(@in.string, "unix")

//output
"2025-10-06T20:24:24Z"
                    
//input
{"milli": "1257894000000"}

//program
SET @out = parse_time(@in.milli, "unix_milli")

//output
"2009-11-10T23:00:00Z"
                    
//input
{"micro": "1257894000000000"}

//program
SET @out = parse_time(@in.micro, "unix_micro")

//output
"2009-11-10T23:00:00Z"
                    
//input
{"nano": "1257894000000000000"}

//program
SET @out = parse_time(@in.nano, "unix_nano")

//output
"2009-11-10T23:00:00Z"
                    
//input

//program

//output
"2009-11-10T00:00:00Z"

Higher Order Functions:

filter std.filter(target:MAP|ARRAY, filter_function:ARROW) MAP|ARRAY

Iterates over each entry of the target (map or array), and either keep or remove the entry based on the 'return' key being set to true(keep) or false(remove) in the arrow function

Tags: Higher Order

Params:

target

The target array or map to iterate over

filter_function

The filtering arrow function. The original entry value will either be kept (true) or discarded (false) based on the value of the "return" variable when the arrow function is finished. The orignal value can be accessed via the ".value" path from the named variable passed to the arrow function. For maps: The key name for an entry is accessible via the ".key" path from the named variable passed to the arrow function; you cannot reassign keys using this function. For arrays: The index number for an entry is accessible via the ".index" path from the named variable passed to the arrow function; you cannot reassign index numbers using this function.

Return:

Description: The map or array containing the newly filtered values

Examples:

//input
[1, "2"]

//program
//filter an array
SET @out.result = filter(@in, e ~> {
	SET return = int(e.value) % 2 == 0
})

//output
{"result": ["2"]}
                    
//input
{"a": 1, "b": "2"}

//program
//filter a map
SET @out.result = filter(@in, e ~> {
	SET return = int(e.value) % 2 == 0
})

//output
{"result": {"b": "2"}}

reduce std.reduce(target:MAP|ARRAY, accumulator:BASIC, reduce_function:ARROW) BASIC

Iterates over each entry of the target (map or array), and updates the accumulator based on the 'return' key being set in the arrow function

Tags: Higher Order

Params:

target

The target array or map to iterate over

accumulator

The value that will eventually be returned by the function after being updated each iteration

reduce_function

The reducer arrow function. The accumulator will be updated based to the value of the "return" variable when the arrow function is finished. The accumulator can be accessed via the ".current" path from the named variable passed to the arrow function. The orignal value can be accessed via the ".value" path from the named variable passed to the arrow function. For maps: The key name for an entry is accessible via the ".key" path from the named variable passed to the arrow function; you cannot reassign keys using this function. For arrays: The index number for an entry is accessible via the ".index" path from the named variable passed to the arrow function; you cannot reassign index numbers using this function.

Return:

Description: The map or array containing the newly filtered values

Examples:

//input
[1, "2", "buckle my shoe"]

//program
//reduce an array of numbers to sum its contents
SET @out.result = reduce(@in, 0, e ~> {
	SET cleaned_number = int(e.value) | catch(0) // these variables are ONLY set locally to the arrow function environment.
	SET return = e.current + cleaned_number
})

//output
{"result": 3}
                    
//input
{"a": 1, "b": "2"}

//program
//reduce a map to sum the values.
SET @out.result = reduce(@in, NULL, e ~> {
	IF e.current == NULL :: SET e.current = 0
	SET cleaned_number = int(e.value) | catch(0)
	SET return = e.current + cleaned_number
})

//output
{"result": 3}
